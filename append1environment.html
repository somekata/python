付録1: Pythonの環境構築

この付録では、Pythonを使い始めるための環境構築について説明します。Python自体をインストールする方法から、AnacondaやBiocondaといったパッケージ管理ツールを使った方法、GitHubやGitBashの利用、そして仮想環境の設定方法までを網羅しています。

1. Pythonのインストール

Python単体のインストール

Pythonは公式サイト（https://www.python.org/downloads/）から直接インストールできます。最新バージョンのインストーラーをダウンロードし、ガイドに従ってインストールしてください。インストール時には「PATHに追加する」オプションを忘れずに選択してください。

Anacondaを使ったインストール

Anaconda は、Pythonとその関連ライブラリ、ツールを一括で管理できる便利なディストリビューションです。データサイエンスや機械学習の環境構築に最適です。Anacondaのインストールは、公式サイトからインストーラーをダウンロードし、指示に従ってインストールします。

Biocondaを使ったインストール

Bioconda は、バイオインフォマティクス関連のツールを簡単にインストール・管理できるコンダ・チャンネルです。AnacondaやMinicondaをインストールした後に、Biocondaを利用することでバイオインフォマティクス向けのパッケージが手軽に使えるようになります。Biocondaの利用方法については、公式ドキュメント（Bioconda）を参考にしてください。

2. GitHubの利用

GitHub は、ソフトウェアのバージョン管理や共同作業を行うためのプラットフォームです。Pythonプロジェクトのソースコード管理に役立ちます。GitHubを使うことで、プロジェクトの履歴管理やチームでの共同作業が容易になります。GitHubのアカウントを作成し、リポジトリを作成してプロジェクトを公開・管理しましょう。

GitHubの使い方の基本:

  • リポジトリの作成: プロジェクトごとにリポジトリを作成し、ソースコードを管理します。
  • コミット: 変更を保存し、履歴として記録します。
  • プルリクエスト: 変更内容を提案し、レビューを依頼する機能です。

3. GitBashとUbuntuの利用

ターミナル、コマンドプロンプト、PowerShell

開発やコマンド操作に使われるターミナルには、いくつかの種類があります。基本的な操作は似ていますが、プラットフォームや用途によって使い分けが行われます。

  • ターミナル（Terminal）: UNIX/Linux環境やMac OSで標準的に利用されるコマンドラインインターフェースです。BashやZshといったシェルが主に使用され、コマンドを直接入力して操作を実行できます。
  • コマンドプロンプト（Command Prompt）: Windowsの標準的なコマンドラインインターフェースで、基本的なファイル操作やプログラムの実行ができます。PowerShellと比較すると、Windows環境でのシステム管理に使われることが多いです。
  • PowerShell: Windowsに標準で搭載されているシェルで、コマンドプロンプトの拡張版と言えます。強力なスクリプト機能を持ち、システム管理に特化したコマンドやスクリプトが使用できます。

GitBash

GitBash は、Windows環境でもUNIX系のコマンドを使うことができるターミナルエミュレーターです。Gitを利用する際に非常に便利で、Bashコマンドを使ってプロジェクトの管理ができます。GitBashは、こちらからダウンロードしてインストールできます。

Ubuntu

Ubuntu は、人気の高いLinuxディストリビューションであり、特に開発環境として利用されることが多いです。Ubuntuを使用することで、オープンソースツールをフル活用した開発環境が構築できます。公式サイトからダウンロードして、インストールできます。

4. 仮想環境の使い方

Pythonでは、プロジェクトごとに異なるライブラリやバージョンを管理するために仮想環境を使用します。仮想環境はプロジェクトごとに分離されているため、依存関係の競合を防ぎます。

仮想環境の作成

Python標準の`venv`モジュールを使って仮想環境を作成できます。次のコマンドを実行してください。 

python -m venv myenv

上記のコマンドで`myenv`という仮想環境が作成されます。

仮想環境のアクティベート

仮想環境をアクティベートするには、以下のコマンドを使用します。

  • Windowsの場合: myenv\Scripts\activate
  • Mac/Linuxの場合: source myenv/bin/activate

仮想環境の終了

仮想環境を終了するには、以下のコマンドを実行します。

deactivate

仮想環境でのライブラリインストール

仮想環境内で必要なライブラリをインストールする際は、通常通り`pip`を使います。 

pip install パッケージ名

5. VSCodeの使い方

Visual Studio Code (VSCode) は、軽量でありながら強力なエディタで、多くのプログラミング言語に対応しています。PythonやJavaScript、HTML/CSSなど、さまざまな言語をサポートしており、拡張機能を使ってさらに機能を追加することができます。以下では、VSCodeの基本的な使い方を説明します。

VSCodeのインストール

VSCodeは公式サイト（こちら）からダウンロードできます。インストーラーをダウンロードして指示に従ってインストールを完了してください。

VSCodeの基本操作

VSCodeを起動すると、以下のような基本的な機能が使えます。

  • ファイルの作成・編集: エディタ内で新しいファイルを作成し、コードを記述できます。左側の「エクスプローラー」からファイルを開くか、新規作成できます。
  • ターミナル: VSCode内蔵のターミナルを使用して、システムのコマンドや仮想環境の操作が可能です。ターミナルはメニューの「ターミナル」から開けます。
  • コード補完とシンタックスハイライト: PythonやJavaScriptなど、多くのプログラミング言語でシンタックスハイライトやコード補完機能が提供されています。

拡張機能

VSCodeにはさまざまな拡張機能が用意されており、プログラミングの効率を向上させるために役立ちます。以下は、Python開発に役立つ主要な拡張機能です。

  • Python拡張機能: Pythonコードの実行やデバッグ、仮想環境の自動検出をサポートする必須の拡張機能です。拡張機能はVSCodeの「拡張機能」タブから検索してインストールできます。
  • Pylint: Pythonコードのスタイルやエラーをチェックするためのツールです。
  • Jupyter: VSCode内でJupyter Notebookを開いて実行できる拡張機能です。

デバッグ機能

VSCodeは強力なデバッグツールを提供しており、コードのエラーを見つけて修正するのに役立ちます。Pythonの場合、次の手順でデバッグが可能です。

  1. VSCode内でPythonファイルを開きます。
  2. 左側の「Run and Debug」アイコンをクリックし、デバッグモードを開始します。
  3. ブレークポイントを設定して、コードをステップ実行することができます。

VSCodeのカスタマイズ

VSCodeは自分の好みに合わせてカスタマイズできます。例えば、テーマを変更したり、キーボードショートカットを自分用に設定することが可能です。「設定」メニューからさまざまなカスタマイズオプションにアクセスできます。
